This mod requires Minecraft Forge. The minimum required build is listed next to each build in the Download section.

To install on a Windows client machine, place the mod zip or jar file into your Minecraft directory under the mod folder, typically located at: %APPDATA%/.minecraft/mods/

To install on a server, place the mod zip or jar file into the server mod folder.

If you're having difficulty installing the mod, avi12 has provided video installation instructions.

Installing the Mod

Migrating 1.6.X worlds to 1.7.X

World migration code was first introduced in build v2.1.2 for MC 1.6.4, and build v3.2.0 for MC 1.7.2. For best results, don't attempt to migrate a world using versions of this mod that are several releases apart. It is recommended you use v3.2.8 on both ends when trying to migrate worlds.

To migrate your MC 1.6.4 worlds to 1.7.2 or greater, follow these steps:

  1. Load Carpenter's Blocks for MC 1.6.4 into the folder of the same version in your mods directory.
  2. Load Carpenter's Blocks for MC 1.7.2 or 1.7.10 into the folder of the same version in your mods directory.
  3. Start the game in MC 1.6.4 and load the world.
  4. Ensure all chunks with Carpenter's Blocks have been loaded into memory by visiting all locations where the blocks are used.
  5. Exit game.

The blocks should now be prepared for migration. Start the game in MC 1.7.2 or 1.7.10 and load the world. If the blocks don't migrate properly, try using a later version of Forge (build 1121 is the minimum recommended).

What won't survive the migration:

  • Multi-block entities like placed doors and beds may disappear, so it's recommended that you remove them first.
  • Chisel patterns and designs will disappear from placed blocks.

My game crashed! What do I do?

You can either post it in spoilers in the official thread if not already reported (do your best to search first), or as a new issue here if it doesn't already exist.

Chisel designs, fancy water, ice, colored glass.. none of it renders correctly in 1.7.2 and up, why?

Minecraft releases in the 1.7 range began sorting quads that rendered in the alpha pass, which required making many adjustments to the mod. It has been resolved in version 3.2.7.

Will you include a localization file for my region?

Submit a pull request to the Github repository in the navigation bar at the top to have your language file(s) added.

To load your own language files right away, simply drop the .lang file into the zip or jar in the folder assets/carpentersblocks/lang/ and they will be loaded at runtime.

Why won't game NPCs detect barriers, doors, garage doors, etc? And why won't leads tie to Carpenter's Barrier posts?

Many aspects of Minecraft are hard-coded to certain objects, like horse leads only surviving when tied to vanilla fence posts, and NPCs only detecting vanilla doors. It has been determined that any solution to this problem would require a heavy amount of core changes, and is not within the scope of this mod.

I'm having issues with this mod and MCPatcher, why?

This bug affected older builds primarily for Minecraft version 1.6. The latest builds in the 3.0.0 range are no longer affected.

Power isn't propagating through the Carpenter's Block. Why not?

Before version v3.3.0, this was not possible without rendering side-effects. It has now been added and works only for the original Carpenter's Block.
